## FYP Timeline Overview

The project follows UITU's FYP milestone structure across two semesters (Capstone 1 and 2). Each milestone has specific deliverables evaluated by the FYP panel.

```
Semester 1 (Capstone 1)
  Week 1-6  ─── Proposal + Research + Non-Working Prototype
  Week 7    ─── MILESTONE 1: Architecture + Prototype Review
  Week 8-13 ─── POC Development
  Week 14   ─── MILESTONE 2: POC Demonstration

Semester 2 (Capstone 2)
  Week 1-6  ─── Full System Implementation
  Week 7    ─── MILESTONE 3: Working System Demo
  Week 8-13 ─── Integration, Testing, Polish
  Week 14   ─── MILESTONE 4: Final Defense + Report
```

### What's Built So Far

**✅ Market Data Simulator (Complete)**
- 5 venue servers running on ports 8001-8005
- GBM with regime-switching for realistic price simulation
- Configurable personality profiles per venue
- V3 (GammaMarkets) configured as degraded/blacklisted
- FastAPI endpoints: `/quote`, `/orderbook`, `/health`, `/execute-order`, `/metrics`
- Admin endpoints: `/admin/degrade`, `/admin/recover`

**✅ Market Data Aggregator (Complete)**
- Collects quotes from all 5 venues periodically
- Computes NBBO (best bid/offer across all venues)
- Displays venue table with health status
- V3 correctly shows BLACKLIST status

**✅ Frontend Prototype (Complete — Figma Make)**
- React + TypeScript + Vite project generated by Figma Make
- 18+ screens mapped in the application
- Dark theme trading dashboard aesthetic
- POC screens identified: Login, Dashboard, Venue Connectivity, Risk Manager, Kill Switch

**🔄 Backend Services (In Progress)**
- Gateway API structure defined
- Need to build: Routing Engine, Risk Engine, Order State Manager
- Need to connect: Frontend ↔ Backend via REST + WebSocket

**⬜ Not Yet Started**
- RAFT Consensus implementation
- Venue Health Monitor (as separate service — currently baked into aggregator)
- Policy Engine
- Database schema + migrations
- Integration testing

### POC Development Plan

The POC is NOT a throwaway. It is Phase 1 of the full system. The code, structure, and architecture established now will carry through to Milestones 3 and 4. We are building for keeps.

#### Phase 1: Backend Foundation (POC Scope)

```
Week 1-2: Core Infrastructure
├── Project scaffolding (monorepo structure)
├── Shared models (Pydantic: Order, Venue, Quote, Risk)
├── Database setup (PostgreSQL schema + Redis)
├── Gateway API skeleton (FastAPI + routers)
└── Environment configuration (.env, Docker)

Week 3-4: Service Implementation
├── Market Data Service (migrate existing simulator)
├── Venue Health Monitor (extract from aggregator)
├── Order State Manager (state machine)
├── Risk Engine (pre-trade checks + kill switch)
└── Routing Engine (best price strategy)

Week 5-6: Integration
├── Frontend ↔ Backend connection (REST + WebSocket)
├── WebSocket real-time streaming
├── End-to-end order flow demo
├── Kill switch demonstration
└── POC presentation preparation
```

## Milestone 3 — Full Working System (Planned)

**Date:** Semester 2, Week 7
**Scope:** Complete implementation of all system features

### Target Deliverables

1. **Complete Routing Engine** — All 3 strategies (BestPrice, LiquiditySweep, VWAP)
2. **Full Risk Engine** — Position tracking, notional exposure, rate limiting
3. **RAFT Consensus** — 3-node cluster with leader election and log replication
4. **Policy Engine** — Configurable rule evaluation with automated actions
5. **Complete Frontend** — All 18 screens fully connected to backend
6. **Multi-Cloud Deployment** — Venues on AWS/GCP/Azure, platform on primary cloud
7. **FIX Protocol Integration** — QuickFIX sessions with venue simulators
8. **Comprehensive Testing** — Unit (80%+), integration, load testing
